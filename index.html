Research Project 01

All-Terrain Autonomous Vehicle Research

This project compares passive, active, and semi-active suspension control strategies for a Baja-scale autonomous all-terrain vehicle using a quarter-car vertical dynamics model built and simulated in MATLAB, together with suspension design work in SolidWorks.

Project overview

I studied how different suspension control strategies affect ride comfort, chassis stability, and packaging constraints for an autonomous ATV platform. The work focused on a 2-DOF quarter-car model evaluated with both a step-bump input and a washboard sinusoidal road profile, while also supporting the project with a double wishbone front suspension assembly in SolidWorks.

Skills used

  • MATLAB
  • SolidWorks
  • Vehicle dynamics
  • ODE45 time-domain simulation
  • Active and semi-active control logic
  • Technical report writing

Simulation results

These MATLAB response plots compare passive, active, and semi-active suspension behavior across body acceleration, suspension travel, and tire deflection during the step-bump test.

Team collaboration

This work was part of the Autonomous Vehicle Laboratory at Cal Poly Pomona, where the suspension analysis supported a larger vehicle-development effort. My modeling and report work connected directly to the platform's broader design and stability goals.

My contributions

I built the quarter-car model, implemented active actuator saturation and semi-active skyhook-style damping logic, simulated the suspension response in MATLAB, compared RMS body acceleration and maximum suspension travel, designed a double wishbone front suspension assembly in SolidWorks, and documented the results in a formal technical report.

Key findings

The active suspension performed best overall for the ATV platform. On the step bump, it kept maximum suspension travel to 0.191 ft compared to 0.223 ft for the semi-active case while maintaining nearly identical RMS body acceleration. On the washboard input, semi-active control produced the lowest RMS acceleration, but the active system still required the least suspension travel.

MATLAB Project 05

Lotka-Volterra Predator-Prey Simulation

This project used MATLAB and `ode45` to simulate the coupled Lotka-Volterra predator-prey equations over a 35-year period and compare how parameter changes affect population behavior.

Project overview

The project examined how predator and prey populations evolve over time under the Lotka-Volterra model. In addition to the baseline case, a sensitivity analysis was performed by doubling one model parameter at a time to study changes in peak populations, oscillation timing, and system response.

Skills used

  • MATLAB
  • Simulation
  • Mathematical modeling
  • Data interpretation
  • ODE45 numerical integration
  • Sensitivity analysis

Team collaboration

This project was completed with Gabriel Phillips as part of a mechanical engineering computation course, with the report and simulation work developed around a shared modeling assignment.

My contributions

I helped build the MATLAB simulation structure, define the predator-prey rate-of-change function, run multiple `ode45` solutions, generate comparison plots, and interpret how parameter changes affected oscillation size, timing, and predator-prey relationships.

Key findings

The baseline case produced periodic oscillatory behavior with peak populations of 318 prey and 158 predators and an approximate cycle length of 7.4 years. Parameter changes significantly altered peak values, minimum values, and the timing of the predator-prey cycles.

MATLAB Project 06

Vibration and Temperature Analysis of a 4-Pole Motor

This project analyzed vibration and temperature data from a four-pole electric motor using FFT-based frequency analysis, order identification, and moving-average filtering to isolate the motor's real dynamic and thermal behavior.

Project overview

The goal of this project was to identify the motor's running speed, harmonic content, sub-running-speed vibration, and peak surface temperature from recorded test data. The analysis focused on separating true motor behavior from measurement noise and interference.

Skills used

  • MATLAB
  • FFT signal analysis
  • Bandpass filtering
  • Order and harmonic analysis
  • Thermal thinking
  • Moving-average filtering
  • Engineering interpretation

Team collaboration

This was completed as an individual project for a mechanical engineering computation course, with the analysis, MATLAB implementation, and interpretation developed independently.

My contributions

I processed the vibration signal in MATLAB using FFT, identified the dominant running-speed frequency and harmonic orders, applied a bandpass filter around the running-speed region, filtered the noisy thermocouple signal with a moving average, and reported the final vibration and temperature results.

Key findings

The dominant vibration frequency was 59.4 Hz, corresponding to a running speed of about 3564 rpm with an amplitude of 2.467 g. The filtered motor surface temperature reached 45.44 C, and the frequency spectrum showed a strong 4x harmonic plus a 0.5x sub-harmonic that pointed toward mechanical looseness and misalignment-related behavior.

Interpretation

The analysis suggested that the 1x peak aligned with rotor unbalance, the strong 2x and 4x components suggested possible misalignment, and the 0.5x component was consistent with mechanical looseness rather than random electrical noise alone.
